## What This Document Is NOT

This is **not** a simplification of the physics. The Teleodynamic ML document uses physics formalism because the phenomena it describes are genuinely geometric/topological, not merely metaphorical.

However, engineers implementing the system do not need to solve Hamiltonians. They need to:
1. Implement the flows (Night Cycle, ledger sorting, centroid evolution)
2. Respect the rhythms (Day/Night distinction, periodic integration)
3. Preserve the structure (tensegrity, sacred boundaries)

The physics describes the *why*. This document bridges to the *how*. Neither replaces the theoretical understanding — they serve different purposes.
